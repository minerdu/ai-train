# AI培训 APP 代码开发计划 V3

更新时间：2026-04-27

## 1. 当前开发原则

本轮先完成产品文档与开发计划修订，确认后再进行代码改造。

代码实现必须遵守：

- `ai-fran-app` 仍是 AI培训 APP 的整体界面、结构、交互和工程基线。
- `AI-OPS-APP` 的 Skill中心、SOP建模、指标预警、AI自主运行、日报、文档、账号、接入和登录设置可继续复用。
- AI培训的业务差异集中在培训内容、实战陪跑、角色权限和AI运营只读信号，不重新发明一套UI。

## 2. V3信息架构

原结构：

```text
任务｜训练｜AI培训｜实战｜我的
```

调整为：

```text
任务｜实战训练｜AI培训｜角色｜我的
```

调整原因：

- 当前 `训练` 和 `实战` 内容差异度不足，继续拆分会导致页面和数据重复。
- 大部分模拟训练应由真实业务问题触发，并进入同一个实战训练闭环。
- AI培训与AI招商、AI运营最大的差异是员工、店长、总部三类用户的任务、页面、数据和AI指令权限不同，因此需要独立一级 `角色` Tab。

## 3. 页面改造计划

### Phase 1：导航与路由

- 底部Tab改为 `任务｜实战训练｜AI培训｜角色｜我的`。
- 将现有 `训练` 与 `实战` 能力合并到新的一级页面。
- 新增 `角色` 一级页面。
- 底部图标改为培训业务语义，保持系列产品风格但不与AI招商完全相同。

### Phase 2：实战训练页面

使用现有训练页面的布局和UI作为基础，重做内容：

- `核心训练`：门店盈利模型、30天落地、六大流程、咨询诊断、手法、品项卡项、5A体验、顾客经营。
- `实战陪跑`：从AI运营只读信号生成真实业务场景，例如新客未办卡、老客未到店、疗程快用完、投诉、B档案缺失。
- `复盘提升`：根据陪练分、实战问题、店长审核生成补练。

### Phase 3：角色页面

新增员工、店长、总部三类视图：

- 员工：我的任务、我的实战训练、我的能力画像、我的AI培训。
- 店长：本店员工、任务完成、弱项、作业审核、晨会带教、门店周报。
- 总部：多店Skill、SOP、指标、权限、AI运营接入、抽检和效果复盘。

权限必须覆盖：

- 页面可见。
- 数据范围。
- AI指令范围。
- 任务创建权限。
- 审核和抽检责任。
- AI运营只读信号可见范围。

### Phase 4：Skill内容替换

按 `产品开发文档/ai_training_skill_catalog_v2.md` 更新页面数据：

- 30天门店落地训练Skill。
- 六大服务流程标准化Skill。
- 咨询诊断与需求洞察Skill。
- 手法与项目专业通关Skill。
- 品项与卡项体系Skill。
- 店长带教与晨会复盘Skill。
- 顾客经营361与B档案Skill。
- 5A体验与五感场景Skill。
- 总部Skill发布与SOP建模Skill。

### Phase 5：AI培训中枢适配

- 快捷指令增加“生成实战训练”“按AI运营信号补练”“查看角色权限”“生成店长晨会”“生成总部抽检计划”。
- AI回复按员工、店长、总部角色返回不同数据范围和动作。
- AgentRun过程展示读取角色、读取Skill、读取AI运营只读信号、生成任务、等待审核等步骤。

### Phase 6：我的/设置延续

保持上一轮已确定的设置结构：

- 系统状态：AI大模型、知识库、AI运营系统、Skill中心。
- AI智能体配置：AI教练设置、AI大模型与知识库、红线与安全规则、群组智能体。
- 标杆企业Skill设置：Skill中心、培训SOP建模、指标与预警。
- AI培训设置：AI自主培训引擎、培训日报、培训文档。
- 运营管理：账号管理、AI运营系统接入、登录选项、权限管理。

## 4. 代码实现顺序

1. 先改导航、路由和页面入口，不调整业务细节。
2. 再合并训练/实战页面，使用静态V3数据验证结构。
3. 再新增角色页面和权限矩阵。
4. 再替换Skill内容和AI培训快捷指令。
5. 最后接入AI运营只读模拟信号，验证实战陪跑链路。
